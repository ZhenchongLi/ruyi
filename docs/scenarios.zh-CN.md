## 引言

Ruyi（如意）与 Claude Code 的结合，提供了一种结构化的 AI 辅助开发模式。与单次调用 AI 生成代码不同，如意引入了**双代理审查**机制：Agent A（实现者）完成工作后，由独立的 Agent B（审查者）进行对抗性审查。两者互不可见对方的推理过程，如意作为裁判决定提交、修订还是回滚。这种机制确保了产出质量，避免了 AI 生成代码中常见的"看起来对但实际有问题"的情况。

除了双代理审查，如意还提供了两项关键保障。**Worktree 隔离**——每个任务在独立的 git worktree 中运行，不会影响你的工作目录，每个子任务要么原子性提交，要么完整回滚，不会留下半成品。**可恢复执行**——任务被中断后，重跑 `ruyi do` 会自动跳过已完成的子任务，从断点继续。这三项特性共同构成了如意的安全契约：你可以放心地把复杂任务交给它，不用担心搞乱代码库。

以下六个场景展示了如何在实际工作中利用这些能力。每个场景包含适用情境、完整命令、预期效果和注意事项，你可以直接参照使用。
